Recording Name - Name of the recording. This will be a directory on the device sdcard, usually in /sdcard/Documents/ARRecorder.

Resolutions - Select the camera resolution for the recording

Rotation/Orientation Sensor - Select the type of sensor used for detecting rotation/orientation.

Max Size - Specify the maximum file size used during recording (append G for Gigabytes, M for Megabytes or nothing for bytes)

Record Sensors - Specify sensors to record data for.

Options

Off-Device Finalization - Only perform initial raw frame and sensor recording on the Android device. A desktop postprocessor available at https://github.com/donaldmunro/AARemu can then be used to perform the frame conversion and post-processing much faster than is possible on an Android device, particularly if the desktop has a modern GPU so CUDA or OpenCL can be used to accelerate the processing.

Flash On - Switch the camera flash on when recording (only visible if the device has a flash)

Stitching - Use image stitching to provide a 3-up averaging filter for rotational recordings (only available for rotational recording). Note this will take a long time on most Android devices so the use of Off-Device Finalization is recommended (if Off-Device Finalization is enabled then this setting does not make any difference as stitching is a command line option in the post-processor).

Camera2 API - Use the Camera2 API if available (for this application there is no real advantage to using the Camera2 API)
